EXIT键行为设计

确定短按和长按EXIT键的不同响应

问题：在快速设置菜单中，EXIT键的短按和长按应该如何处理？

A

短按返回MENU，长按直达IDLE（推荐）

行为:
• EXIT短按: QUICK_SET → HOME_MENU（返回上级菜单）
• EXIT长按: QUICK_SET → HOME_IDLE（直接退出所有菜单）

导航流程图
HOME_IDLE（主界面-无光标）
↓ UP/DOWN唤醒
HOME_MENU（主界面-光标显示）
↓ ENTER "快速设置"
QUICK_SET（快速设置子菜单）
✓ EXIT短按 → 返回 HOME_MENU
用户可能想选择"系统设置"而不是"快速设置"
⚡ EXIT长按 → 直达 HOME_IDLE
快速退出所有菜单，回到待机状态

优点

  • 层级清晰: 短按逐级返回，符合菜单导航直觉
  • 长按直达: 提供快捷退出，避免多次按键
  • 与主界面一致: HOME_MENU短按也是返回IDLE
  • 灵活性高: 用户可以返回上级选择其他功能

缺点

  • 需要用户理解短按/长按的区别
当前代码（state_machine.c:160-170）:
// QUICK_SET: 短按返回MENU
if (s_ui_state == UI_STATE_QUICK_SET) {
    s_ui_state = UI_STATE_HOME_MENU;
    return;
}

// QUICK_SET: 长按直达IDLE
if (s_ui_state == UI_STATE_QUICK_SET) {
    s_ui_state = UI_STATE_HOME_IDLE;
    return;
}

B

短按和长按都直达IDLE

行为:
• EXIT短按: QUICK_SET → HOME_IDLE
• EXIT长按: QUICK_SET → HOME_IDLE（相同）

优点

  • 简化逻辑，短按长按效果相同
  • 符合"快速"定位（快进快出）

缺点

  • 灵活性差: 无法返回上级菜单选择其他功能
  • 不符合层级导航原则: 破坏了菜单的层级结构
  • 如果用户误入快速设置，无法优雅地返回选择系统设置
C

仅短按有效，长按无特殊功能

行为:
• EXIT短按: QUICK_SET → HOME_MENU（或HOME_IDLE）
• EXIT长按: 无动作

优点

  • 最简单的实现

缺点

  • 浪费长按功能: 无法提供快捷退出
  • 如果短按返回IDLE，则无法返回MENU（问题同B）
  • 如果短按返回MENU，则缺少快速退出方式

✅ 推荐：方案A - 短按返回MENU，长按直达IDLE

这是最符合人机交互原则的设计：

  • 符合层级导航: 短按逐级返回是标准做法
  • 提供快捷方式: 长按直达避免多次按键
  • 代码已实现: state_machine.c:160-170和183-186已有此逻辑
  • 与整体设计一致: 其他子菜单（SYS_SET）也是相同逻辑
