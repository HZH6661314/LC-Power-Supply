确定用户按键操作后的系统响应

问题：用户按ENTER确认预设后，应该如何处理？

A

应用预设并返回主界面（推荐）

流程: ENTER确认 → 立即应用电压/电流设定 → 自动返回HOME_IDLE状态 → 用户可以直接按SET开启输出

优点

  • 效率高: 一键完成，减少操作步骤
  • 符合"快速"定位: 快速设置就是为了快
  • 用户心理模型清晰: 选择即应用
  • 代码简单: 不需要额外的确认状态

缺点

  • 误触风险（但可通过EXIT长按返回缓解）
  • 无法在应用前预览/微调
状态机流程:
QUICK_SET → (ENTER) → 调用SM_ApplyQuickSet() → s_ui_state = HOME_IDLE
B

应用预设但停留在菜单

流程: ENTER确认 → 应用设定 → 保持在QUICK_SET状态 → 用户需要手动按EXIT返回

优点

  • 用户可以连续尝试多个预设
  • 提供"反悔"机会（再选其他预设）

缺点

  • 不符合"快速"定位: 需要额外操作
  • 用户可能困惑："为什么还在这个界面？"
  • 增加交互复杂度
C

应用预设后进入编辑模式

流程: ENTER确认 → 应用设定 → 进入HOME_EDIT状态 → 用户可以微调后再确认

优点

  • 预设作为"起点"，可微调
  • 灵活性高

缺点

  • 最不符合"快速"定位
  • 交互路径最复杂
  • 如果用户只想用预设值，反而增加操作
  • 状态跳转逻辑复杂（QUICK_SET → HOME_EDIT需要额外处理焦点）
