光标移动逻辑

确定UP/DOWN键移动光标的边界行为

问题：当光标在第一项时按UP，或在最后一项时按DOWN，应该如何处理？

A

循环滚动（推荐）

行为: 光标在第一项时按UP → 跳到最后一项；光标在最后一项时按DOWN → 跳到第一项

循环示意
预设1: 5V/1A
预设2: 9V/2A
预设3: 12V/3A
预设4: 20V/3A
⟳
光标可以从任意位置回到起点

优点

  • 符合用户直觉: 常见于各类菜单系统
  • 操作连贯: 不会"卡"在边界上
  • 导航效率高: 可以快速到达任意项（最多按3次）
  • 代码已实现: state_machine.c:237-243已有循环逻辑

缺点

  • 首次使用可能需要适应（但这是业界标准）
当前代码（state_machine.c:237-243）:
if (s_quick_set_cursor > 0U) {
    s_quick_set_cursor--;
} else {
    s_quick_set_cursor = 3U;  // ✅ 循环到最后
}
B

边界停止

行为: 光标在第一项时按UP → 无动作；光标在最后一项时按DOWN → 无动作

边界停止示意
⛔ 顶部边界
预设2: 9V/2A
预设3: 12V/3A
⛔ 底部边界
⊥
光标在边界处"停止"

优点

  • 边界感明确，不会"迷失"
  • 适合线性浏览（从上到下）

缺点

  • 导航效率低: 从最后回到第一项需要按3次UP
  • 不符合业界惯例: 大多数嵌入式设备菜单都是循环的
  • 需要修改代码: 当前实现是循环逻辑
  • 按键无响应会让用户困惑（"是不是坏了？"）

✅ 推荐：循环滚动

基于以下原因，强烈推荐循环滚动：

  • 代码已经实现，无需修改
  • 符合用户对嵌入式设备菜单的普遍预期
  • 提高操作效率（4项菜单，循环最多按3次，边界停止最差按3次）
  • 与主界面的焦点移动逻辑一致（state_machine.c:414-425也是循环）
