## 快速开始

### 1. 初始化

在使用W25Q256之前，必须先初始化BSP层和驱动：

```c
#include "bsp_spi.h"
#include "bsp_gpio.h"
#include "W25Q256.h"

int main(void)
{
    // 1. 硬件初始化
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    
    // 2. 初始化BSP层
    BSP_SPI_Init();   // 初始化SPI
    BSP_GPIO_Init();  // 初始化GPIO
    
    // 3. 初始化W25Q256驱动
    W25Q256_Init();
    
    // 4. 自检（可选）
    if (W25Q256_SelfTest()) {
        printf("W25Q256 初始化成功！\n");
    } else {
        printf("W25Q256 初始化失败！\n");
    }
    
    while (1) {
        // 你的应用代码
    }
}
```

### 2. 基本读写流程

```c
// 写入数据的基本流程
uint8_t write_data[256] = {0x01, 0x02, 0x03, ...};

// 1. 擦除扇区（必须先擦除才能写入）
W25Q256_EraseSector(0x00000000);

// 2. 写入数据
W25Q256_WriteData(0x00000000, write_data, 256);

// 3. 读取数据
uint8_t read_data[256];
W25Q256_ReadData(0x00000000, read_data, 256);

// 4. 验证数据
for (int i = 0; i < 256; i++) {
    if (read_data[i] != write_data[i]) {
        printf("数据错误！\n");
    }
}
```

## API参考

### 初始化和自检

#### `void W25Q256_Init(void)`
**功能：** 初始化W25Q256驱动  
**参数：** 无  
**返回：** 无  
**说明：** 
- 必须在使用其他API之前调用
- 会自动进入4字节地址模式
- 会唤醒Flash（如果处于掉电模式）

**示例：**
```c
W25Q256_Init();
```

---

#### `uint8_t W25Q256_SelfTest(void)`
**功能：** 自检，检查Flash是否正常工作  
**参数：** 无  
**返回：** 1=正常，0=异常  
**说明：** 通过读取JEDEC ID来判断Flash是否正常

**示例：**
```c
if (W25Q256_SelfTest()) {
    printf("Flash正常\n");
} else {
    printf("Flash异常\n");
}
```

---

### 读取ID

#### `void W25Q256_ReadID(uint8_t *manufacturer_id, uint16_t *device_id)`
**功能：** 读取Flash的制造商ID和设备ID  
**参数：**
- `manufacturer_id`: 制造商ID（输出参数，应为0xEF）
- `device_id`: 设备ID（输出参数，应为0x4019）

**返回：** 无

**示例：**
```c
uint8_t mfr_id;
uint16_t dev_id;
W25Q256_ReadID(&mfr_id, &dev_id);
printf("制造商ID: 0x%02X, 设备ID: 0x%04X\n", mfr_id, dev_id);
// 输出：制造商ID: 0xEF, 设备ID: 0x4019
```

---

#### `void W25Q256_ReadUniqueID(uint8_t *unique_id)`
**功能：** 读取Flash的唯一ID（64位）  
**参数：**
- `unique_id`: 唯一ID缓冲区（8字节）

**返回：** 无  
**说明：** 每个Flash芯片都有唯一的64位ID

**示例：**
```c
uint8_t uid[8];
W25Q256_ReadUniqueID(uid);
printf("唯一ID: ");
for (int i = 0; i < 8; i++) {
    printf("%02X ", uid[i]);
}
printf("\n");
```

---

### 读取数据

#### `void W25Q256_ReadData(uint32_t addr, uint8_t *buffer, uint32_t len)`
**功能：** 读取数据（普通读取）  
**参数：**
- `addr`: 读取地址（0x00000000 ~ 0x01FFFFFF，32MB）
- `buffer`: 数据缓冲区
- `len`: 读取长度

**返回：** 无  
**说明：** 
- 支持全地址空间（32MB）
- 可以跨页、跨扇区读取
- 读取速度约50MHz

**示例：**
```c
uint8_t buffer[1024];
W25Q256_ReadData(0x00001000, buffer, 1024);  // 从地址0x1000读取1KB数据
```

---

#### `void W25Q256_FastRead(uint32_t addr, uint8_t *buffer, uint32_t len)`
**功能：** 快速读取数据  
**参数：** 同`W25Q256_ReadData()`  
**返回：** 无  
**说明：** 
- 比普通读取快（约80MHz）
- 需要发送1个dummy字节
- 适合大量数据读取

**示例：**
```c
uint8_t buffer[4096];
W25Q256_FastRead(0x00000000, buffer, 4096);  // 快速读取4KB数据
```

---

### 写入数据

#### `void W25Q256_WritePage(uint32_t addr, const uint8_t *buffer, uint32_t len)`
**功能：** 写入一页数据（256字节以内）  
**参数：**
- `addr`: 写入地址
- `buffer`: 数据缓冲区
- `len`: 写入长度（最大256字节）

**返回：** 无  
**说明：** 
- 一次最多写入256字节
- 不能跨页写入
- 写入前必须先擦除

**示例：**
```c
uint8_t data[256] = {0x01, 0x02, 0x03, ...};
W25Q256_EraseSector(0x00000000);  // 先擦除
W25Q256_WritePage(0x00000000, data, 256);  // 写入一页
```

---

#### `void W25Q256_WriteData(uint32_t addr, const uint8_t *buffer, uint32_t len)`
**功能：** 写入数据（自动处理跨页）  
**参数：**
- `addr`: 写入地址
- `buffer`: 数据缓冲区
- `len`: 写入长度（无限制）

**返回：** 无  
**说明：** 
- 自动处理跨页写入
- 写入前必须先擦除
- 适合写入大量数据

**示例：**
```c
uint8_t data[1024] = {0x01, 0x02, 0x03, ...};
W25Q256_EraseSector(0x00000000);  // 先擦除
W25Q256_WriteData(0x00000000, data, 1024);  // 写入1KB，自动跨页
```

---

### 擦除操作

#### `void W25Q256_EraseSector(uint32_t addr)`
**功能：** 擦除扇区（4KB）  
**参数：**
- `addr`: 扇区地址（任意地址，自动对齐到4KB边界）

**返回：** 无  
**说明：** 
- 擦除时间：约45-400ms
- 擦除后数据全部变为0xFF
- 写入前必须先擦除

**示例：**
```c
W25Q256_EraseSector(0x00001234);  // 擦除包含0x1234的扇区（0x1000-0x1FFF）
```

---

#### `void W25Q256_EraseBlock64K(uint32_t addr)`
**功能：** 擦除块（64KB）  
**参数：**
- `addr`: 块地址（任意地址，自动对齐到64KB边界）

**返回：** 无  
**说明：** 
- 擦除时间：约150-2000ms
- 比擦除16个扇区快
- 适合擦除大块数据

**示例：**
```c
W25Q256_EraseBlock64K(0x00010000);  // 擦除64KB块
```

---

#### `void W25Q256_EraseChip(void)`
**功能：** 擦除整个芯片（32MB）  
**参数：** 无  
**返回：** 无  
**说明：** 
- 擦除时间：约20-100秒
- 会擦除所有数据
- 谨慎使用！

**示例：**
```c
W25Q256_EraseChip();  // 擦除整个芯片（需要很长时间）
```

---

### 电源管理

#### `void W25Q256_PowerDown(void)`
**功能：** 进入掉电模式（降低功耗）  
**参数：** 无  
**返回：** 无  
**说明：** 
- 功耗从5mA降低到1uA
- 掉电模式下无法读写
- 需要调用`W25Q256_WakeUp()`唤醒

**示例：**
```c
W25Q256_PowerDown();  // 进入掉电模式
```

---

#### `void W25Q256_WakeUp(void)`
**功能：** 唤醒（退出掉电模式）  
**参数：** 无  
**返回：** 无  
**说明：** 
- 唤醒时间：约3us
- 唤醒后可以正常读写

**示例：**
```c
W25Q256_WakeUp();  // 唤醒
```

---

#### `void W25Q256_WaitBusy(void)`
**功能：** 等待Flash空闲  
**参数：** 无  
**返回：** 无  
**说明：** 
- 在擦除/写入操作后自动调用
- 一般不需要手动调用

## 使用示例

### 示例1：读取Flash ID

```c
void Example_ReadID(void)
{
    uint8_t mfr_id;
    uint16_t dev_id;
    
    W25Q256_ReadID(&mfr_id, &dev_id);
    
    if (mfr_id == 0xEF && dev_id == 0x4019) {
        printf("W25Q256检测成功！\n");
    } else {
        printf("Flash ID错误：0x%02X 0x%04X\n", mfr_id, dev_id);
    }
}
```

---

### 示例2：写入和读取数据

```c
void Example_WriteRead(void)
{
    uint8_t write_data[256];
    uint8_t read_data[256];
    
    // 1. 准备数据
    for (int i = 0; i < 256; i++) {
        write_data[i] = i;
    }
    
    // 2. 擦除扇区
    printf("擦除扇区...\n");
    W25Q256_EraseSector(0x00000000);
    
    // 3. 写入数据
    printf("写入数据...\n");
    W25Q256_WriteData(0x00000000, write_data, 256);
    
    // 4. 读取数据
    printf("读取数据...\n");
    W25Q256_ReadData(0x00000000, read_data, 256);
    
    // 5. 验证数据
    int errors = 0;
    for (int i = 0; i < 256; i++) {
        if (read_data[i] != write_data[i]) {
            errors++;
        }
    }
    
    if (errors == 0) {
        printf("数据验证成功！\n");
    } else {
        printf("数据验证失败！错误数：%d\n", errors);
    }
}
```

### 示例3：存储参数

```c
// 参数结构体
typedef struct {
    float voltage_set;    // 设定电压
    float current_set;    // 设定电流
    uint8_t mode;         // 工作模式
    uint32_t checksum;    // 校验和
} SystemParams_t;

#define PARAMS_ADDR  0x00000000  // 参数存储地址

// 保存参数
void SaveParams(const SystemParams_t *params)
{
    // 计算校验和
    SystemParams_t temp = *params;
    temp.checksum = 0;
    uint32_t sum = 0;
    uint8_t *p = (uint8_t*)&temp;
    for (int i = 0; i < sizeof(SystemParams_t) - 4; i++) {
        sum += p[i];
    }
    temp.checksum = sum;
    
    // 擦除并写入
    W25Q256_EraseSector(PARAMS_ADDR);
    W25Q256_WriteData(PARAMS_ADDR, (uint8_t*)&temp, sizeof(SystemParams_t));
    
    printf("参数已保存\n");
}

// 加载参数
int LoadParams(SystemParams_t *params)
{
    // 读取参数
    W25Q256_ReadData(PARAMS_ADDR, (uint8_t*)params, sizeof(SystemParams_t));
    
    // 验证校验和
    uint32_t sum = 0;
    uint8_t *p = (uint8_t*)params;
    for (int i = 0; i < sizeof(SystemParams_t) - 4; i++) {
        sum += p[i];
    }
    
    if (sum == params->checksum) {
        printf("参数加载成功\n");
        return 1;
    } else {
        printf("参数校验失败\n");
        return 0;
    }
}

// 使用示例
void Example_SaveLoadParams(void)
{
    SystemParams_t params;
    
    // 保存参数
    params.voltage_set = 12.5f;
    params.current_set = 3.0f;
    params.mode = 1;
    SaveParams(&params);
    
    // 加载参数
    SystemParams_t loaded_params;
    if (LoadParams(&loaded_params)) {
        printf("电压: %.2fV, 电流: %.2fA, 模式: %d\n", 
               loaded_params.voltage_set, 
               loaded_params.current_set, 
               loaded_params.mode);
    }
}
```

### 示例4：存储多组预设

```c
#define PRESET_COUNT  10
#define PRESET_BASE_ADDR  0x00001000  // 预设起始地址
#define PRESET_SIZE  256  // 每个预设256字节

// 保存预设
void SavePreset(uint8_t index, const uint8_t *data, uint32_t len)
{
    if (index >= PRESET_COUNT || len > PRESET_SIZE) {
        printf("参数错误\n");
        return;
    }
    
    uint32_t addr = PRESET_BASE_ADDR + (index * PRESET_SIZE);
    
    // 擦除扇区（每个扇区4KB，可以存储16个预设）
    if (index % 16 == 0) {
        W25Q256_EraseSector(addr);
    }
    
    // 写入数据
    W25Q256_WriteData(addr, data, len);
    
    printf("预设%d已保存\n", index);
}

// 加载预设
void LoadPreset(uint8_t index, uint8_t *data, uint32_t len)
{
    if (index >= PRESET_COUNT || len > PRESET_SIZE) {
        printf("参数错误\n");
        return;
    }
    
    uint32_t addr = PRESET_BASE_ADDR + (index * PRESET_SIZE);
    W25Q256_ReadData(addr, data, len);
    
    printf("预设%d已加载\n", index);
}
```

### 示例5：数据日志记录

```c
#define LOG_BASE_ADDR  0x00100000  // 日志起始地址（1MB）
#define LOG_SIZE  (16 * 1024 * 1024)  // 日志区域大小（16MB）
#define LOG_ENTRY_SIZE  64  // 每条日志64字节

static uint32_t log_write_addr = LOG_BASE_ADDR;

// 写入日志
void WriteLog(const uint8_t *data, uint32_t len)
{
    if (len > LOG_ENTRY_SIZE) {
        len = LOG_ENTRY_SIZE;
    }
    
    // 检查是否需要擦除扇区
    if ((log_write_addr % 4096) == 0) {
        W25Q256_EraseSector(log_write_addr);
    }
    
    // 写入日志
    W25Q256_WriteData(log_write_addr, data, len);
    log_write_addr += LOG_ENTRY_SIZE;
    
    // 循环写入
    if (log_write_addr >= (LOG_BASE_ADDR + LOG_SIZE)) {
        log_write_addr = LOG_BASE_ADDR;
    }
}

// 读取日志
void ReadLog(uint32_t index, uint8_t *data, uint32_t len)
{
    uint32_t addr = LOG_BASE_ADDR + (index * LOG_ENTRY_SIZE);
    W25Q256_ReadData(addr, data, len);
}
```

## 常见问题

### Q1: 为什么写入的数据读不出来？

**A:** 写入前必须先擦除！Flash的特性是只能将1变为0，不能将0变为1。擦除操作会将所有位变为1（0xFF）。

```c
// ❌ 错误：没有擦除
W25Q256_WriteData(0x00000000, data, 256);

// ✅ 正确：先擦除再写入
W25Q256_EraseSector(0x00000000);
W25Q256_WriteData(0x00000000, data, 256);
```

---

### Q2: 擦除操作需要多长时间？

**A:** 
- 扇区擦除（4KB）：45-400ms
- 块擦除（64KB）：150-2000ms
- 芯片擦除（32MB）：20-100秒

擦除时间会自动等待，不需要手动延时。

---

### Q3: 地址范围是多少？

**A:** W25Q256是256Mbit（32MB），地址范围：
- 起始地址：0x00000000
- 结束地址：0x01FFFFFF
- 总容量：33,554,432字节

---

### Q4: 一次最多能写入多少数据？

**A:** 
- `W25Q256_WritePage()`：最多256字节（一页）
- `W25Q256_WriteData()`：无限制（自动跨页）

---

### Q5: Flash的寿命是多少？

**A:** W25Q256的擦写次数：
- 典型值：100,000次擦写循环
- 数据保持：20年

建议使用磨损均衡算法延长寿命。

---

### Q6: 如何提高读写速度？

**A:** 
1. 使用`W25Q256_FastRead()`代替`W25Q256_ReadData()`
2. 使用块擦除代替多次扇区擦除
3. 批量写入数据，减少片选切换次数

## 注意事项

### ⚠️ 重要提示

1. **写入前必须擦除**
   - Flash只能将1变为0，不能将0变为1
   - 擦除操作将所有位变为1（0xFF）

2. **擦除对齐**
   - 扇区擦除：4KB对齐
   - 块擦除：64KB对齐

3. **写入对齐**
   - 页写入：256字节对齐
   - 不能跨页写入（使用`W25Q256_WriteData()`自动处理）

4. **地址范围**
   - 有效地址：0x00000000 ~ 0x01FFFFFF
   - 超出范围会导致错误

5. **电源管理**
   - 掉电模式下无法读写
   - 使用前必须唤醒

6. **并发访问**
   - 不支持多线程同时访问
   - 需要添加互斥锁保护
