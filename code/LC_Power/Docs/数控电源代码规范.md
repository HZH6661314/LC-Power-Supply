# 100W 数控电源项目编码规范 (V1.3) 🛠️

## 1. 软件架构层级与铁律 (Architecture & Iron Laws) 🧱

本工程采用严格的分层架构。为了保证代码的高可靠性与可维护性，必须将以下“架构铁律”刻在脑子里：

- **铁律一：严格的单向依赖与隔离**
  - **上层可以调用下层，下层绝对不能知道上层的存在。** * 允许：`UI` (Product层) 读取 `Control` 层的电压变量来显示。
    - 严禁：`Driver` 层的 ADC 中断触发时，直接调用 `state_machine.c` 里的函数改变系统状态。
  - **同层之间禁止直接通信。** 必须通信时，使用“全局状态结构体（Context）”作为媒介。
    - 例：`protection.c` 发现过压，决不能直接调用 `ui.h` 去弹窗，而是修改全局状态 `SysStatus.fault_code = OVER_VOLTAGE`。UI 模块在自己的 50ms 轮询中读到该状态，自行决定如何重绘界面。
- **铁律二：平台移植测试（试金石）**
  - **设计标准：** 假设明天 STM32F334 缺货，必须换用 TI C2000 DSP。
  - **合格线：** 只需要删掉并重写 `4_Driver` 文件夹。`3_Control` 文件夹内的 `pid_voltage.c` **绝对不允许有哪怕一行代码的修改**！控制层内部只能包含纯 C 语言的数学加减乘除，严禁出现 `stm32f33x_hal.h` 等任何底层头文件。
- **铁律三：数据的跨层传递（消灭 extern）**
  - **难点：** Control 层的 PID 算出了新的 PWM 占空比，但它不能调用底层的 `HRTIM_SetDuty(duty)`。
  - **解法：** 使用“函数参数与返回值”进行依赖注入。调度器读取底层数据传入算法，算法返回计算结果，调度器再将结果下发给底层执行。严禁使用 `extern` 导致全局变量漫天飞。

## 2. 目录架构与六层依赖模型 🧱

严格遵守**单向依赖**原则，各层职责如下：

| **层级**          | **职责说明**            | **典型组件**                                                 | **依赖关系**                               |
| ----------------- | ----------------------- | ------------------------------------------------------------ | ------------------------------------------ |
| **0_SysCore**     | **系统调度引擎 (新增)** | **SysTick时间片轮询、全局中断配置、临界区并发保护**          | **统筹全局调度，可调用下方各层初始化接口** |
| **1_Product**     | 最终业务实现            | UI显示、上位机协议、参数存储调度                             | 可调用 App, Control, Driver                |
| **2_Application** | 系统逻辑与安全          | 全局状态机(OFF/ON切换)、软启动算法、保护逻辑、PD诱骗握手(Boot阶段) | 可调用 Control, Driver                     |
| **3_Control**     | 数学核心（纯C）         | 电压环 PID、电流环 2P2Z、各类软件滤波器                      | **严禁依赖任何底层头文件**                 |
| **4_Driver**      | 外部组件驱动            | W25Q256 驱动、OLED 驱动、INA240 参数换算                     | 仅可调用 BSP 层接口                        |
| **5_BSP**         | 芯片级板级支持          | STM32F334 HAL库、HRTIM底层初始化、ADC/DMA 配置、SPI 底层收发 | 最底层，无依赖                             |

## 3. 核心状态机定义 (State Machine) 🔌

- **`STATE_BOOT`:** 上电瞬间进入，执行系统初始化与 PD 28V 诱骗握手。完成后自动跳转 `STATE_STANDBY`。
- **`STATE_STANDBY` (输出 OFF):** 主功率管关闭，允许参数配置、Flash 读写、UI 界面交互。
- **`STATE_SOFT_START`:** 过渡状态，开启功率管并平滑拉升占空比，防浪涌。
- **`STATE_RUNNING` (输出 ON):** 控制环路全速工作，持续供电。
- **`STATE_FAULT`:** 触发保护后锁定输出，等待用户复位

## 4. 命名契约 (Naming Convention) 🏷️

- **模块前缀:** 所有函数必须带有模块前缀，例如：
  - `Drv_Adc_GetValue()`
  - `App_State_SwitchTo()`
- **变量可见性:** * **局部变量:** `camelCase` (例: `currentStep`)。
  - **私有全局变量 (static):** `s_` 开头 (例: `s_is_calibrated`)。
  - **公有全局变量:** `g_` 开头 (例: `g_sys_context`) —— **限制使用**。
- **物理单位:** 涉及物理量的变量名必须后缀单位，如 `voltage_mv`, `current_ma`, `temp_c`。

## 5. 时间域管理 (Task Scheduling) ⚡

系统任务按紧急程度划分，严禁跨域干扰：

1. **极速域 (Isr_Critical):** HRTIM/ADC 中断。仅处理环路计算、更新 PWM 占空比、紧急限流。耗时须 < 1.5us。
2. **实时域 (Task_1ms):** SysTick 触发。执行状态机、软件保护检查、PID 目标值平滑增减（软启动）。
3. **异步域 (Task_Main):** 主循环轮询。执行屏幕刷新、Flash 写入、串口调试。

## 6. 异常处理规范 (Safety & Error) 🚨

- **函数返回:** 非极速域函数必须返回状态码（如 `RESULT_OK`, `RESULT_ERR_TIMEOUT`）。
- **硬保护:** 凡涉及过压、过流、短路等足以炸管的异常，**必须**在中断内第一时间调用 `Drv_Hrtim_EmergencyStop()`，严禁等待主循环处理。
- **状态同步:** 进入 `STATE_FAULT` 后，必须锁定输出，直到用户手动复位。
