## 第一阶段：架构设计三大铁律（2周，理论为主）

### 模块1：铁律一 - 严格的单向依赖与隔离

**核心原则：**

1. **上层可以调用下层，下层绝对不能知道上层的存在**
2. **同层之间禁止直接通信，必须通过全局Context**

**理论讲解：**

**为什么需要单向依赖？**

- 防止循环依赖（A调B，B调A，代码改不动）
- 保证可移植性（底层换芯片，上层不受影响）
- 降低耦合度（修改一个模块不影响其他模块）

**正确示例：**

```c
// ✅ 允许：Product层读取Control层数据
void UI_Display_Voltage(void) {
    int32_t voltage = g_sys_context.real_voltage_mv;
    TFT_DrawNumber(voltage);
}
```

**错误示例：**

```c
// ❌ 严禁：Driver层ADC中断直接调用Application层
void ADC_IRQHandler(void) {
    uint32_t adc_value = ADC1->DR;
    App_State_SwitchTo(STATE_FAULT);  // 错误！底层不能调上层
}
```

**正确做法：**

```c
// ✅ ADC中断只处理数据
void ADC_IRQHandler(void) {
    g_sys_context.adc_raw_value = ADC1->DR;
}

// Application层在1ms任务中检查
void Task_1ms(void) {
    if (g_sys_context.adc_raw_value > OVP_THRESHOLD) {
        App_State_SwitchTo(STATE_FAULT);
    }
}
```

**同层通信规则：**

```c
// ❌ 错误：protection.c直接调用ui.c
void Protection_Check(void) {
    if (over_voltage) {
        UI_ShowFaultDialog();  // 错误！同层直接调用
    }
}

// ✅ 正确：通过全局Context通信
void Protection_Check(void) {
    if (over_voltage) {
        g_sys_context.fault_code = FAULT_OVP;
    }
}

void UI_Task_50ms(void) {
    if (g_sys_context.fault_code != FAULT_NONE) {
        UI_ShowFaultDialog();
    }
}
```

**实战练习：**

1. 绘制你项目的依赖关系图
2. 检查代码中是否有违反单向依赖的地方
3. 设计全局Context结构体（PowerSystemContext_t）
4. 重构一个模块，消除跨层调用

### 模块2：铁律二 - 平台移植测试（试金石）

**核心原则：** 假设明天STM32F334缺货，必须换用TI C2000 DSP。

- **合格线：** 只需要删掉并重写`4_Driver`和`5_BSP`文件夹
- **黄金标准：** `3_Control`文件夹内的`pid_voltage.c`**绝对不允许有哪怕一行代码的修改**

**理论讲解：**

**Control层的纯净性要求：**

```c
// ✅ 正确：Control层只有纯C数学运算
float PID_Calculate(float target, float current) {
    float error = target - current;
    float output = Kp * error + Ki * integral + Kd * derivative;
    return output;
}

// ❌ 错误：Control层出现硬件相关代码
float PID_Calculate(float target, float current) {
    float error = target - current;
    HAL_GPIO_WritePin(LED_GPIO_Port, LED_Pin, GPIO_PIN_SET);  // 错误！
    return Kp * error;
}

// ❌ 错误：Control层包含硬件头文件
#include "stm32f3xx_hal.h"  // 错误！严禁包含底层头文件
```

**正确的数据流动：**

```c
// Scheduler层协调数据流动
void Task_1ms(void) {
    // 1. 从底层读取数据
    float voltage = BSP_ADC_GetVoltage();
    
    // 2. 传入Control层计算
    float duty = PID_Voltage_Calculate(g_target_voltage, voltage);
    
    // 3. 下发给底层执行
    BSP_HRTIM_SetDuty(duty);
}
```

1. 检查Control层代码，是否包含任何硬件相关头文件
2. 尝试在PC上编译Control层代码（使用gcc）
3. 思考：如果要移植到Arduino，哪些文件需要重写？

### 模块3：铁律三 - 数据的跨层传递（消灭extern）

**核心原则：** 使用"函数参数与返回值"进行依赖注入，严禁使用extern导致全局变量漫天飞。

**理论讲解：**

**错误示例：extern满天飞**

```c
// pid_voltage.c (Control层)
extern uint32_t g_adc_voltage;  // 错误！
extern void HRTIM_SetDuty(uint16_t duty);  // 错误！

float PID_Calculate(void) {
    float voltage = g_adc_voltage * 0.001f;
    float duty = Kp * (target - voltage);
    HRTIM_SetDuty((uint16_t)duty);
    return duty;
}
```

**正确示例：依赖注入**

```c
// pid_voltage.c (Control层) - 纯函数
float PID_Calculate(float target, float current) {
    float error = target - current;
    s_integral += error * dt;
    float derivative = (error - s_last_error) / dt;
    s_last_error = error;
    
    return Kp * error + Ki * s_integral + Kd * derivative;
}

// task_manager.c (Scheduler层)
void Task_1ms(void) {
    float voltage = BSP_ADC_GetVoltage() / 1000.0f;
    float duty = PID_Calculate(g_target_voltage, voltage);
    
    // 限幅
    if (duty > 0.95f) duty = 0.95f;
    if (duty < 0.05f) duty = 0.05f;
    
    BSP_HRTIM_SetDuty((uint16_t)(duty * 1000));
}
```

**全局Context的正确使用：**

```c
// state_machine.h
typedef struct {
    int32_t  target_voltage_mv;
    int32_t  target_current_ma;
    int32_t  real_voltage_mv;
    int32_t  real_current_ma;
    SystemState_t state;
    FaultCode_t   fault_code;
    uint16_t pwm_duty;
} PowerSystemContext_t;

extern PowerSystemContext_t g_sys_context;  // 唯一允许的全局变量
```

**实战练习：**

1. 设计PowerSystemContext_t结构体
2. 重构一个函数，消除extern，改用依赖注入
3. 统计项目中有多少个extern变量，制定消除计划

## 第二阶段：时间域管理与状态机（3周，理论+实战）

### 模块4：时间域管理（Task Scheduling）

**核心概念：** 系统任务按紧急程度划分为三个域，严禁跨域干扰：

1. **极速域 (Isr_Critical):** HRTIM/ADC中断
   - 仅处理环路计算、更新PWM占空比、紧急限流
   - 耗时须 < 1.5us
2. **实时域 (Task_1ms):** SysTick触发
   - 执行状态机、软件保护检查、PID目标值平滑增减
3. **异步域 (Task_Main):** 主循环轮询
   - 执行屏幕刷新、Flash写入、串口调试

**理论讲解：**

**为什么要划分时间域？**

- 保证实时性（紧急任务不被慢速任务阻塞）
- 降低CPU占用率（不同优先级任务分时执行）
- 避免任务冲突（快速任务和慢速任务隔离）

**时间域划分原则：**

```c
// 极速域：中断内处理（< 1.5us）
void ADC_IRQHandler(void) {
    uint32_t adc_value = ADC1->DR;
    
    // 硬保护：立即停止PWM
    if (adc_value > CRITICAL_THRESHOLD) {
        Drv_Hrtim_EmergencyStop();
    }
}

// 实时域：1ms任务
void Task_1ms(void) {
    StateMachine_Process();
    Protection_SoftCheck();
    PID_Control();
}

// 异步域：主循环
void Task_Main(void) {
    UI_Refresh();
    Flash_Write();
    UART_Process();
}
```

**实战练习：**

1. 分析你的项目，哪些任务属于哪个时间域？
2. 测量每个任务的执行时间（使用DWT）
3. 优化任务调度，降低CPU占用率
4. 实现任务执行时间监控和告警

### 模块5：状态机设计

**核心概念：** 状态机是嵌入式系统的核心，定义了系统的行为模式。

**五态状态机：**

```
STATE_BOOT (上电初始化)
    ↓ PD握手完成
STATE_STANDBY (待机，输出OFF)
    ↓ 用户按下输出按钮
STATE_SOFT_START (软启动)
    ↓ 占空比平滑拉升完成
STATE_RUNNING (运行，输出ON)
    ↓ 触发保护
STATE_FAULT (故障锁定)
    ↓ 用户手动复位
STATE_STANDBY
```

**理论讲解：**

**状态机的实现方式：**

```c
// 方式1：Switch-Case（简单但不优雅）
void StateMachine_Process(void) {
    switch (current_state) {
        case STATE_STANDBY:
            // 处理待机逻辑
            break;
        case STATE_RUNNING:
            // 处理运行逻辑
            break;
    }
}

// 方式2：函数指针表（推荐）
typedef struct {
    void (*Enter)(void);
    void (*Run)(void);
    void (*Exit)(void);
} StateHandler_t;

StateHandler_t g_state_handlers[] = {
    [STATE_STANDBY] = { Standby_Enter, Standby_Run, Standby_Exit },
    [STATE_RUNNING] = { Running_Enter, Running_Run, Running_Exit },
};

void StateMachine_SwitchTo(SystemState_t new_state) {
    if (g_state_handlers[current_state].Exit) {
        g_state_handlers[current_state].Exit();
    }
    
    current_state = new_state;
    
    if (g_state_handlers[current_state].Enter) {
        g_state_handlers[current_state].Enter();
    }
}
```

1. 绘制完整的状态转移图
2. 实现五个状态的Enter/Run/Exit函数
3. 实现STATE_SOFT_START的占空比平滑拉升算法
4. 添加状态转移日志，便于调试

## 第三阶段：控制算法与保护机制（4周，实战为主）

### 模块6：电压环PID控制

**理论知识：** PID控制是最常用的闭环控制算法，由比例(P)、积分(I)、微分(D)三部分组成。

**PID公式：**

```
output = Kp * error + Ki * ∫error*dt + Kd * d(error)/dt
```

**实战练习：**

1. 实现基础PID算法
2. 添加积分限幅（防止积分饱和）
3. 添加微分滤波（降低噪声影响）
4. 参数整定（Ziegler-Nichols方法）
5. 单元测试（Python生成测试数据）

**代码实现：**

```c
// pid_voltage.c
typedef struct {
    float Kp;
    float Ki;
    float Kd;
    float integral;
    float last_error;
    float integral_max;
} PID_t;

float PID_Calculate(PID_t *pid, float target, float current) {
    float error = target - current;
    
    // 积分项
    pid->integral += error * DT;
    if (pid->integral > pid->integral_max) {
        pid->integral = pid->integral_max;
    }
    if (pid->integral < -pid->integral_max) {
        pid->integral = -pid->integral_max;
    }
    
    // 微分项
    float derivative = (error - pid->last_error) / DT;
    pid->last_error = error;
    
    // 输出
    return pid->Kp * error + pid->Ki * pid->integral + pid->Kd * derivative;
}
```

### 模块7：电流环2P2Z控制

**理论知识：** 2P2Z（2极2零点）补偿器是数字电源中常用的控制算法，比PID更适合高频控制。

**差分方程：**

```
y[n] = b0*x[n] + b1*x[n-1] + b2*x[n-2] - a1*y[n-1] - a2*y[n-2]
```

**实战练习：**

1. 理解2P2Z补偿原理

### 模块8：保护机制设计

**核心概念：** 保护机制分为硬保护和软保护。

**硬保护（中断内处理）：**

```c
void ADC_IRQHandler(void) {
    uint32_t current = ADC_GetCurrent();

        Drv_Hrtim_EmergencyStop();

```

```c
void Protection_Process(void) {

        g_sys_context.pwm_duty *= 0.9f;
    }
}
```
